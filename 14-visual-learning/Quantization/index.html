Step 1 — Scalar Quantization
Scalar quantization is the simplest form — just use fewer bits per number. A 32-bit float becomes an 8-bit integer by mapping the range [min, max] to [0, 255]. You lose some precision but the math stays the same.
Bit depth 8 bits
32-bit16-bit8-bit4-bit
Recap: Scalar quantization trades bits for precision. 8-bit gives ~4× compression with ~3% recall loss — a great default choice.
Step 2 — Binary Quantization
Binary quantization is the most aggressive form of scalar quantization. Each float becomes a single bit — just the sign (≥0 → 1, <0 → 0). This gives 32× compression but works best with high-dimensional vectors (512+ dims).
32× smaller
32 bits per float → 1 bit per float

✅ Good fit

  • Vectors with 512+ dimensions
  • Large indexes (1M+ vectors)
  • Memory is the bottleneck

❌ Poor fit

  • Low-dimensional vectors (<128 dims)
  • Small datasets that fit in RAM
  • Recall accuracy is critical
Recap: Binary = 1 bit per dimension = 32× smaller. Works great with 768+ dim embeddings like OpenAI or BERT.
Step 3 — Product Quantization (PQ)
Product Quantization splits each vector into M subvectors. For each sub-space, run k-means to find representative centroids. Store only the centroid ID instead of the original floats. The set of centroids for each sub-space is called a codebook.
K-means clustering on a 2D sub-space
PQ parameters 3 subvectors × 16 centroids (4 bits)
2 sub3 sub4 sub5 sub6 sub
Recap: PQ splits vectors, clusters subspaces, and stores centroid IDs. It achieves 16-256× compression with tunable accuracy — the choice of production vector databases.
Step 4 — Residual Quantization (RQ)
Residual Quantization refines the approximation iteratively. After the first quantization pass, compute the residual (the error). Then quantize that residual too. Each stage adds a correction layer, progressively reducing the reconstruction error.
Residual stages 3 stages
1 stage234
Recap: RQ = iterative refinement. Each residual stage corrects the previous error. 3 stages of 8-bit gives ~98% recall at 3× the cost of single-stage scalar.
Step 5 — Additive Quantization (AQ)
Additive Quantization represents each vector as the sum of codewords from multiple codebooks. Unlike PQ which splits the vector into independent subspaces, AQ lets all codebooks contribute to all dimensions. This gives more flexibility — two codebooks with 256 centroids each can represent 65,536 combinations.
AQ parameters 2 codebooks × 16 centroids (8 bits)
2 CB3 CB4 CB
Recap: AQ = multiple cooperating codebooks. More expressive than PQ at the same bit rate. Used in Faiss and state-of-the-art compression research.

Key Takeaway

There is no single best quantization

The right choice depends on your vector dimensions, dataset size, accuracy requirements, and engineering capacity. Scalar 8-bit is the safest default. PQ is the production favorite. RQ gives the best accuracy.

Summary — key takeaways
  • Scalar (8-bit): Best default — 4× smaller, ~97% recall, zero complexity.
  • Binary (1-bit): Maximum simplicity — 32× smaller, needs 512+ dims for good recall.
  • Product Quantization: Best compression — up to 256×, production favorite with tunable accuracy.
  • Residual Quantization: Best accuracy — iterative refinement achieves ~99% recall.
  • Additive Quantization: Most expressive — cooperating codebooks, research-grade quality.
  • All methods trade some recall for massive memory savings. The key is choosing the right tradeoff for your use case.
