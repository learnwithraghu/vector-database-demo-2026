# Plan: Build Interactive RAG Educational Page "The Open-Book Brain"

## Overview
Create a single self-contained HTML file `open-book-brain.html` that teaches Retrieval-Augmented Generation (RAG) through a 5-stage interactive experience. The page follows strict no-scroll design principles from the html-design-guide skill and uses analogy-driven teaching from the vectordb-instructor skill.

## Project Structure
- **Output**: `lessons/open-book-brain.html` (self-contained, no external dependencies)
- **Framework**: Vanilla HTML/CSS/JS only
- **Aesthetic**: Dark Cosmic palette with Syne/DM Sans fonts
- **Navigation**: Stage-based with progress dots and Next buttons
- **Animations**: CSS transitions, typewriter effects, drag-drop interactions

## Stage Breakdown

### Stage 0: Hook ("Even the smartest brain forgets.")
- **Layout**: Centered pulsing brain icon (CSS keyframe animation)
- **Content**: Headline + sub-line text
- **Interaction**: Single "Let's find out →" button
- **Unlock**: Immediate (button click)

### Stage 1: LLM Alone (Closed-Book Test)
- **Layout**: Chat-like UI with question bubble and AI response bubble
- **Animation**: Typewriter effect for AI response (JS implementation)
- **Content**: Hallucinated answer with "No source. Pure fabrication." badge
- **Unlock**: After typewriter completes (~3-4 seconds)

### Stage 2: The Vault (Choose Your Chunk)
- **Layout**: 2x2 grid of document chunk cards + drop zone below
- **Interaction**: HTML5 drag-and-drop API
  - Draggable chunks with visual feedback
  - Only Chunk A (correct) accepts drop
  - Wrong chunks bounce back with CSS shake animation
- **Content**: 4 realistic document snippets
- **Unlock**: Only after correct chunk (A) is successfully dropped

### Stage 3: LLM + Context (Open-Book Answer)
- **Layout**: Split-screen design
  - Left: Highlighted chunk text
  - Right: Chat UI with new AI response
- **Animation**: Typewriter for new response + arrow animation from chunk to AI
- **Content**: Grounded answer citing the chunk with green "Grounded" badge
- **Unlock**: After typewriter completes

### Stage 4: The RAG Pipeline (How It Works)
- **Layout**: Horizontal pipeline diagram
- **Animation**: Sequential fade-in of pipeline nodes (0.4s delays)
- **Content**: 6-node diagram with brief labels
- **Unlock**: After all nodes appear (~3 seconds total)

### Stage 5: End/Recap
- **Layout**: Centered takeaways with staggered animations
- **Content**: 3 bullet points + closing line
- **Interaction**: "Explore Again" button to restart from Stage 0

## Technical Implementation

### Core Architecture
- **Stage Management**: JS object tracking current stage, stage elements, progress dots
- **Navigation Logic**: `goTo(index)` function with CSS class transitions
- **Unlock System**: Condition checking for each stage before enabling Next button

### Key Animations & Interactions
1. **Stage Transitions**: CSS `translateX` + `opacity` with `transition: 0.4s ease`
2. **Typewriter Effect**: JS function appending characters with `setTimeout`
3. **Drag & Drop**:
   - `dragstart`, `dragover`, `drop` events
   - Visual feedback with CSS classes
   - Shake animation for wrong drops (CSS keyframes)
4. **Pipeline Animation**: Sequential `opacity` changes with staggered timeouts
5. **Pulsing Brain**: CSS `@keyframes` for scale/brightness
6. **Arrow Flow**: CSS animation along a path (optional, or simple fade)

### Fonts & Assets
- Google Fonts import for Syne and DM Sans
- No external images (use CSS for brain icon and arrows)
- Inline SVG preferred for any illustrations

### Responsive Design

- Flexible padding: `clamp(1.5rem, 8vw, 6rem)`
- Text sizes scale appropriately

## Potential Challenges & Solutions

### Drag-Drop Implementation
- **Challenge**: Cross-browser compatibility, visual feedback
- **Solution**: Use modern HTML5 API with fallbacks, CSS for visual states

### Timing & Sequencing
- **Challenge**: Coordinating multiple animations (typewriter, pipeline, fades)
- **Solution**: Promise-based animation sequences or callback chains

### Content Fit
- **Challenge**: Ensuring all content fits 100vh without overflow
- **Solution**: Test with fixed viewport, use flexbox centering, reduce content if needed

### Browser Compatibility
- **Challenge**: file:// protocol, drag-drop support
- **Solution**: Modern browsers only, graceful degradation for older browsers
